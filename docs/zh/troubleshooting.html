🔧 疑难排查

遇到问题时按下面的清单排查。90% 的问题能在前两节解决。

1. 加载项未出现 / 任务窗格空白

  • 检查 manifest.xml 是否正确加载：office-addin-manifest validate manifest.xml
  • 检查 WEF 扫描器：如果使用公司 Managed Installer，要求 Version="1.0.0.0" + AID + SourceLocation 必须严格匹配，三者缺一不可。
  • 检查 dev server：npm start 后确认 https://localhost:3000/taskpane.html 可访问。
  • 证书问题：office-addin-dev-certs install 重装证书。

2. "加载项错误" 弹窗

通常是开发服务器没启，或端口被占用。

lsof -i :3000
# 杀掉占用进程:
kill -9 <PID>
# 重启:
npm start

3. AI 请求失败

401 Unauthorized

  • 检查 API Key 是否正确（前后空格、复制完整）。
  • 检查端点 URL 是否对应你账户的模型提供方。

429 Too Many Requests

  • 账户配额耗尽。登录 platform.deepseek.com 查看用量。
  • 公司网关限流。换一个时段重试。

500 / 502 / 503 / 504

  • 服务端临时故障。等几秒重试。
  • 切换模型（chat → reasoner 或反向）。

Network Error / CORS

  • 浏览器 CORS 拒绝。Office 加载项里的 fetch 通常允许跨域，但自建代理可能拦截 OPTIONS 预检。
  • 公司代理需要把 api.deepseek.com 加入白名单。

4. Ribbon 按钮点击没反应

  • 检查 manifest.xml 里 FunctionFile 是否指向 https://localhost:3000/commands.html。
  • 检查浏览器 DevTools（任务窗格里 F12）控制台是否有错误。
  • 检查 commands.js 是否成功注册所有 ExecuteFunction（控制台应有 "5 commands registered" 之类日志）。

5. 写入失败 / 撤销

"无法解析选区"

  • 选区含合并单元格或非连续区域。
  • 选中完整矩形后再试。

"权限不足"

  • 工作簿是只读或受保护。取消保护后再试。

"PivotTable 创建失败"

  • 检查数据源是否在活动工作表上。
  • 检查表头行是否存在。

6. 日志与诊断

在任务窗格里按 F12 打开 DevTools，查看 Console 标签。常见关键词：

  • DS: stream — 流式事件
  • DS: apply — Apply 阶段
  • DS: error — 错误

遇到 bug 时贴日志可以快速定位。

7. 完全重置

所有设置存在 localStorage，可以清除：

localStorage.clear();
location.reload();
⚠️ 这会清掉所有 API Key 和历史会话。先导出 Markdown 备份。
