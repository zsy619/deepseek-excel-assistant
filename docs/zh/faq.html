❓ 常见问题 (FAQ)

账户与配置

Q: API Key 在哪里填？

任务窗格右上角点 ⚙ 设置，第一项就是 API Key。密钥只存在本地（localStorage），不会上传到任何第三方服务器。

Q: 支持哪些模型 / 端点？

默认走 DeepSeek 官方端点 https://api.deepseek.com/v1/chat/completions，兼容任何 OpenAI 协议的代理。在设置面板的 端点 URL 字段改写即可（例如 Azure OpenAI、自建 Ollama 代理等）。

Q: 怎么用公司内部代理？

把 端点 URL 改成你公司的网关地址，例如 https://llm-proxy.corp.local/v1/chat/completions。模型名也相应改写。

Q: 能用 GPT-4 吗？

只要把端点换成 OpenAI / Azure OpenAI 的兼容地址，模型写 gpt-4o、gpt-4-turbo 等即可。所有 OpenAI 兼容的端点都支持。

Q: 系统提示词怎么写？

设置面板的 系统提示词 字段。建议写成角色 + 行为约束，例如：

你是一名资深财务分析师。
- 回答用中文，给出可验证的 Excel 公式。
- 不要编造数据;不确定时明确说明。

功能相关

Q: AI 回复可以撤销吗？

用 Office 原生的 ⌘ Z 即可撤销单元格写入操作。批量写入（如透视表、报告 sheet）需要手动删除。

Q: 公式诊断一次最多扫多少个？

默认 50 个公式（FORMULA_DIAGNOSIS_LIMIT）。超过会给出建议："选区包含 120 个公式，超出 50 个限制，请缩小范围"。

Q: 透视表创建失败？

常见原因：

  • 选区不是连续矩形（含合并单元格）。
  • 数据列类型混杂（同一列有数字和文本）。
  • Office 旧版（< 2016）不支持 PivotTable API。

Q: 相关性矩阵显示不全？

相关性矩阵要求每列都是数值型。如果某列含文本，AI 会自动跳过并在 reasoning 里说明哪些列被排除。

Q: 列类型推断的置信度低怎么办？

置信度低于 50% 的列 AI 会标记 unknown。可以：

  1. 先清洗数据（去掉混杂类型）。
  2. 增加样例行数（不少于 20 行）。
  3. 点 Apply 仍会应用格式（不阻塞），但可手动调整。

性能与限制

Q: 一次能发多少 token？

取决于所选模型的 context window。DeepSeek-chat 默认 32K。本插件默认：

  • 样例数据：最多 50 行 × 20 列。
  • 单条消息：估算后超出 8K 会自动截断并提示。

Q: 流式输出卡顿？

检查网络（VPN / 公司代理）。也可在设置里把 Temperature 调低到 0.3 让输出更确定性、更短。
